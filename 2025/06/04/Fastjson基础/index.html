setter

第一个for是遍历setter的，里面有四个if判断，成功才能往下走

  1. 名字大于四
  2. 不是静态方法
  3. 返回值是本身类或者空
  4. 参数唯一

ok，过了if判断，中间不重要，我们来到

他会把他加到fieldList里面去，我们看到FieldInfo函数里面去，因为前面说到要把这个函数的getOnly参数变成true

field

这个逻辑和field没什么关系，这里就是遍历field

getter

和setter一样的，要满足一些if

  1. 方法长度大于4
  2. 不是静态方法
  3. 无参数
  4. 返回值类型继承自Collection或Map或AtomicBoolean或AtomicInteger或AtomicLong
  5. 他的参数没有setter方法

我们进到FieldInfo这个方法，和setter是一样的

看到中间的判断，他没有参数，所以会让getOnly = true，ok这样我们就获得了一个不是asm反序列化器，而是一个默认的，这样就能够调试了

小结

记得要用parseObject才能调用getter和setter

parse只能调用setter

可以利用setter方法

  1. 名字大于四
  2. 不是静态方法
  3. 返回值是本身类或者空
  4. 参数唯一

可以利用的getter方法

  1. 方法长度大于4
  2. 不是静态方法
  3. 无参数
  4. 返回值类型继承自Collection或Map或AtomicBoolean或AtomicInteger或AtomicLong
    他的参数没有setter方法
